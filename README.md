# FDD AutoSYS

FDD AutoSYS 是一个面向财务尽调（Financial Due Diligence）的多角色协作方法框架。它的目标不是自动替代财务、审计、税务或投资专业判断，而是把项目启动、证据登记、专题分析、工作底稿、反方复核和人工决策组织成可追踪、可复核的流程。

> 本仓库是公开介绍版，只包含方法和架构说明，不是完整运行包，也不包含客户资料、工作底稿模板、测试数据或内部审阅记录。

## 核心能力

- **Phase -1快速经营画像**：在机械抽表前识别企业的业务模式、业务流、财务流和尽调优先级。
- **证据优先**：项目事实使用`SourceID`追踪，准则和法规使用`KnowledgeSourceID`追踪。
- **专题按需加载**：通过一个财务专题路由按当前问题加载单张方法卡，避免一次性加载全部规则。
- **四角色协作**：区分主代理、执行代理、复核代理和人工决策者的职责。
- **Human Gate**：收入真实性、QoE调整、净债务、税务判断等重大事项必须人工确认。
- **项目工作记忆**：持续记录目标企业的业务和财务特征，并在后续证据与初始判断出现重大偏差时提示用户。
- **便携式发布**：普通项目包保持轻量，开发工具和自动保障能力分层提供。

## 方法架构

```mermaid
flowchart LR
    A["项目资料与用户输入"] --> B["Phase -1经营画像"]
    B --> C["业务流—财务流映射"]
    C --> D["FDD策略与资料索要"]
    D --> E["专题方法卡"]
    E --> F["工作底稿与问题候选"]
    F --> G["反方复核"]
    G --> H["Human Gate"]
    H --> I["阶段摘要与报告草案"]
```

框架坚持三条底线：

1. 文件或解析失败不等于企业风险。
2. 缺少证据不等于已经发现错报。
3. Agent输出是候选和待复核事项，不是正式投资、审计、法律或税务意见。

## 适用场景

- 新项目快速启动与尽调方向判断；
- 报表桥接、收入毛利、研发、QoE、营运资金、净债务和税务专题；
- 多Agent或多工具协作下的任务分工和复核；
- 需要保留证据链、人工决策记录和阶段状态的财务项目。

## 当前公开边界

本仓库不包含完整Skill、Agent定义、registry、工作底稿模板、自动化工具或测试样本。公开可见不代表已授予复制、修改或再分发许可；如后续开放代码或模板，将单独明确版本和许可证。

## 免责声明

本项目仅用于方法研究和流程辅助，不构成投资建议、审计意见、法律意见、税务意见、估值意见或交易建议。具体项目结论应由具备相应经验和授权的专业人员基于完整证据作出。
