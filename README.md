# FDD AutoSYS

FDD AutoSYS 是一个面向科技类企业财务尽调（Financial Due Diligence）的多角色协作Agent工具，适用于Codex等主流Agent工具。包含项目启动、原始证据登记、专题分析、工作底稿、subagent反方复核和人工决策等环节。

> 本仓库是公开介绍版，包含方法和架构说明。

## 核心能力

- **快速经营画像**：Agent在机械抽表作业前快速诊断识别企业的业务模式、业务流、财务流，对数据大额波动和异常值进行重点关注并制定尽调优先级。
- **资料快速建档**：Agent可根据用户上传资料包和网页进行资料快速建档、关键信息提取、结构化和Uid赋值。
- **证据优先**：项目事实使用`SourceID`追踪，准则和法规使用`KnowledgeSourceID`追踪，对话中将给出清晰引用来源。
- **会计准则与IPO差距分析**：准则法规类底层知识库引用《企业会计准则》等权威来源并进行llm_wiki结构化处理，根据主要IPO上市门槛进行差距分析和路径建议。
- **专题Skills按需加载**：采用渐进式披露框架，针对收入确认、研发费用资本化等常见问题通过一个财务专题路由规则按当前问题类型加载对应专题Skills，避免一次性加载全部规则。
- **多角色协作**：明确区分主代理、执行代理、复核代理和人工决策者的职责边界，各Subagent之间采用预置提示词和WP模板进行A2A协作，人工决策主要由对话给出选项和描述。
- **人工审阅**：收入真实性、QoE与报表调整、净债务、税务判断等重大事项必须人工确认。
- **项目工作记忆**：开启尽调项目后Agent持续记录目标企业的业务和财务特征与关键信息，并在后续证据与初始判断出现重大偏差时提示用户。
- **便携式配置**：普通项目包保持轻量，开发工具和自动保障能力分层提供，用户可根据自身习惯选择宿主终端和模型。
- **积木式使用管理**：Agent终端配置相应MCP工具后可进行LDD联查、专利技术分析、可比估值等，用户均可通过自然语言直接给出指令。

## 方法架构

```mermaid
flowchart LR
    A["项目资料与用户输入"] --> B["Phase -1经营画像"]
    B --> C["业务流—财务流映射"]
    C --> D["FDD策略与资料索要"]
    D --> E["专题方法卡"]
    E --> F["工作底稿与问题候选"]
    F --> G["反方复核"]
    G --> H["Human Gate"]
    H --> I["阶段摘要与报告草案"]
```

框架坚持三条底线：

1. 文件或解析失败等问题一律报送人工复核决策。
2. 全部底稿报表附带完整公式。
3. Agent输出是候选和待复核事项，不作为正式投资、审计、法律或税务意见。

## 适用场景

- 新项目快速启动与尽调方向判断；
- 科目余额表与序时账匹配、收入毛利、研发、QoE报表调整、营运资金、净债务、税务专题、财务建模；
- 多Agent或多工具协作下的任务分工和复核；
- 需要保留证据链、人工决策记录和阶段状态的财务项目。
