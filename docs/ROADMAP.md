# 发展路线

## 当前状态

公开介绍版已经形成以下核心设计：

- Phase -1快速经营画像和网络优先的外部研究；
- 业务流—财务流—尽调方向映射；
- 四角色协作和Human Gate；
- SourceID与KnowledgeSourceID证据链；
- 单项目工作记忆和受控跨项目记忆；
- 普通项目包与开发全套包分层；
- 财务专题路由和逐张方法卡验收。

## 已验证专题

### 1. 报表桥接和合并范围

建立主体、期间和报表口径底座，处理调整前后桥接、合并范围和三表勾稽。

### 2. 收入和毛利质量

连接合同、订单、交付、验收、开票、回款和成本结转，分析收入时点、总额净额、量价结构和毛利桥接。

### 3. 研发费用及资本化

连接立项、研究、开发、测试、验收和商业化，区分会计资本化、税务加计扣除和高新技术企业口径。

## 后续专题

按逐张测试和验收的原则继续建设：

1. QoE及调整后EBITDA；
2. 营运资金；
3. 净债务和现金可用性；
4. 税务、政府补助和或有事项；
5. 行业专题和脱敏经验模式。

每张方法卡都必须包含业务特征、会计原理、证据三角、检查程序、替代解释、停止条件、三表和交易影响、资料清单及完整规范性依据。

## 后续保障

- 整合可选自动保障入口；
- 扩充黄金样本和反例测试；
- 完成独立反方复核的发布关闭确认；
- 持续核验会计、税务和监管知识源；
- 根据真实项目反馈改进方法卡，但不写入客户专属结论。

## 正式发布条件

公开说明、普通项目包或开发全套包的“可用”状态必须分别判断。正式版本至少需要：

- 全量测试和严格校验通过；
- 发布允许列表和敏感信息扫描通过；
- 关键计算存在黄金样本和反例；
- 重大复核意见完成整改；
- 独立复核明确确认没有发布阻断项；
- 人工确认版本、范围和专业免责声明。

本路线图描述方法建设方向，不构成固定发布日期或功能承诺。
