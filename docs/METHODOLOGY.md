# 尽调方法

## 1. Phase -1快速经营画像

在大规模OCR、抽表、科目扫描和底稿生成前，先回答：

- 企业如何获客、定价、签约、采购、研发、生产或实施、交付、开票和回款？
- 哪些环节决定收入确认、成本归集、毛利、存货、应收、研发投入和现金消耗？
- 哪些业务特点会影响QoE、营运资金、净债务和估值输入？
- 当前判断来自内部项目证据、外部研究、管理层解释还是分析假设？

网络条件允许时，优先研读目标企业披露、券商或咨询研究、权威机构白皮书、财经科技媒体和重要新闻。外部资料用于形成尽调假设，不能替代目标企业的项目证据。

## 2. 业务流与财务流

将业务活动翻译成可以核验的财务特征：

| 业务环节 | 财务流候选 | 典型尽调方向 |
| --- | --- | --- |
| 获客、报价和签约 | 合同负债、销售费用、客户集中度 | 收入模式、获客成本、关联方 |
| 采购、研发和生产 | 存货、研发费用、开发支出、应付 | 成本归集、资本化、产能利用 |
| 交付和验收 | 收入、合同资产、应收、成本结转 | 收入时点、截止性、毛利 |
| 开票和回款 | 应收、税金、经营现金流 | 回款质量、税务桥接 |
| 售后和退货 | 预计负债、退款负债、返利 | 可变对价、售后成本 |

这一步的目的不是提前下结论，而是快速确定P0、P1和P2尽调方向。

## 3. 主题证据三角

每个重要专题至少检查三面证据：

1. **报表或审计口径**：财务报表、附注、审计调整、TB和GL。
2. **底账或明细口径**：客户、产品、项目、合同、存货、人员、费用或银行明细。
3. **业务、税务或外部支持口径**：合同履约、技术文件、纳税申报、银行流水、函证或权威外部资料。

三面证据应当相互支持。某一面缺失时记录`evidence_gap`；底账无法勾稽时先记录`data_quality_issue`，不能直接升级为错报或风险结论。

## 4. 专题分析统一结构

每张专题方法卡采用相同的十部分：

1. 业务特征；
2. 会计原理；
3. 核心假设；
4. 主题证据三角；
5. 检查程序；
6. 替代解释和反方问题；
7. 机械停止条件；
8. 三表及交易影响；
9. 资料补充清单；
10. 完整规范性依据。

目前已验证的专题包括：

- 报表桥接和合并范围；
- 收入和毛利质量；
- 研发费用及资本化。

后续专题见[发展路线](ROADMAP.md)。

## 5. 机械停止条件

尽调不以“扫描越多越好”为目标。满足以下条件时，应停止无目标扩扫：

- 总额和主要分类已解释；
- 重大异常已形成支持解释、替代解释和证据缺口；
- 主题证据三角达到预设覆盖；
- 剩余事项只能依赖尚未取得的证据；
- 未取得证据已转为定向资料索要或Human Gate问题。

停止条件用于减少低价值重复检查，但不能替代重要性和证据覆盖要求。

## 6. 输出状态

分析输出应明确区分：

- `verified_fact`：已有项目证据支持的事实；
- `external_view`：外部机构或媒体观点；
- `inference`：基于事实形成的推理；
- `hypothesis`：待验证假设；
- `pending_review`：等待复核；
- `human_decision_required`：必须人工确认；
- `external_verification_required`：权威依据或适用性尚待核验。

只有经过规定复核和Human Gate的内容，才能进入正式报告结论。
