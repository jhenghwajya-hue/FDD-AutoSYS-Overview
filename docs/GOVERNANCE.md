# 证据与治理

## 1. 来源追踪

### SourceID

用于追踪项目事实和原始材料。每项重要事实、金额和计算应能回到：

```text
SourceID → 原始文件 → 工作底稿 → Sheet/Cell或证据位置 → 报告表述
```

### KnowledgeSourceID

用于追踪准则、税法、监管规则和权威方法资料。它是内部追踪标识，不能替代面向人的完整引用。

规范性依据首次出现时应列明：

- 发布机关；
- 文件全名；
- 文号；
- 发布或施行日期；
- 适用期间；
- 具体条款或章节；
- 官方来源或本地证据位置。

## 2. Human Gate

下列事项不得由Agent自动关闭：

- 项目范围、主体和报表口径；
- 收入真实性和重大会计判断；
- 重大QoE调整；
- 净债务和debt-like分类；
- 正常化营运资金口径；
- 税务合规、补税或优惠可持续性；
- 重大红旗关闭；
- 正式报告和交易口径。

Agent可以准备事实、计算、候选解释和问题，但最终判断由具备授权的人员确认。

## 3. 错误分类

| 分类 | 示例 | 处理原则 |
| --- | --- | --- |
| `system_error` | 工具、路径、编码或模板失败 | 修复系统或切换方法 |
| `data_quality_issue` | 重复主键、金额不勾稽、字段缺失 | 先修复数据质量 |
| `evidence_gap` | 合同、底账或外部支持未取得 | 转为资料索要 |
| `business_issue` | 经营模式或执行问题候选 | 结合证据进一步分析 |
| `financial_misstatement_risk` | 可能影响确认、计量或列报 | 形成候选并复核 |
| `legal_or_tax_risk` | 可能涉及法规或税务事项 | 引用权威依据并专业复核 |
| `human_decision_required` | 口径或重大结论需人工确定 | 进入Human Gate |

系统失败和资料缺失不能直接写成企业风险。

## 4. 记忆治理

- 单项目工作记忆可以随用户输入、新材料和尽调进展持续更新。
- 重大偏差应向用户报告，并保留变更记录。
- 客户名称、金额、联系方式和项目一次性结论不得进入跨项目正式记忆。
- 可复用方法只有在脱敏、证据核验和人工批量批准后，才能进入正式记忆。

## 5. 隐私和安全

- 原始资料默认只读；
- 派生文件与原始文件分离；
- 外部解析、在线服务或模型调用前检查资料敏感性；
- 不在日志、文档或公开仓库中保存凭据、Token、密钥或个人信息；
- 公开介绍包与完整开发包、项目包分离。

## 6. 质量门

非平凡任务完成前至少检查：

- 文件和必需结构存在；
- schema和registry一致；
- 计算、公式和勾稽通过；
- 关键表述能够回溯证据；
- Human Gate状态没有被误写为已接受；
- 未运行的自动保障明确标记为`assurance_not_run`；
- 发布包不包含越界文件和本机路径。

## 7. 专业边界

FDD AutoSYS 是流程和方法辅助框架，不替代专业人员，也不输出正式投资、审计、法律、税务、估值或交易建议。
